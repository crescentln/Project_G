# Crescentln Ruleset (OpenClash + Surge)

这是你的自建规则仓库，规则由你自己的流水线自动生成和更新。

## 基础地址

- 仓库: [https://github.com/crescentln/new-project](https://github.com/crescentln/new-project)
- Raw Base: `https://raw.githubusercontent.com/crescentln/new-project/main/ruleset/dist`

## 自动更新

- GitHub Actions: `.github/workflows/ruleset-update.yml`
- 计划任务: 每 6 小时自动更新一次

## OpenClash 使用

1. 在 OpenClash 的规则提供器中新增 URL。
2. 直接使用下方“全量规则链接表”的 `OpenClash` 列链接。
3. 推荐优先使用 `non_ip` + `ip` 组合，或按需使用 `domainset`。

示例（AI 规则）:

- non_ip: [ai non_ip](https://raw.githubusercontent.com/crescentln/new-project/main/ruleset/dist/compat/Clash/non_ip/ai.txt)
- ip: [ai ip](https://raw.githubusercontent.com/crescentln/new-project/main/ruleset/dist/compat/Clash/ip/ai.txt)
- domainset: [ai domainset](https://raw.githubusercontent.com/crescentln/new-project/main/ruleset/dist/compat/Clash/domainset/ai.txt)

## Surge 使用

1. 在 Surge 中添加 `RULE-SET` 或 `DOMAIN-SET`。
2. 直接使用下方“全量规则链接表”的 `Surge` 列链接。

示例（AI 规则）:

- RULE-SET(non_ip): [ai non_ip](https://raw.githubusercontent.com/crescentln/new-project/main/ruleset/dist/compat/List/non_ip/ai.conf)
- RULE-SET(ip): [ai ip](https://raw.githubusercontent.com/crescentln/new-project/main/ruleset/dist/compat/List/ip/ai.conf)
- DOMAIN-SET: [ai domainset](https://raw.githubusercontent.com/crescentln/new-project/main/ruleset/dist/compat/List/domainset/ai.conf)

## 策略说明

- `DIRECT`: 直连
- `PROXY`: 走代理
- `REJECT`: 拒绝
- `REJECT-DROP`: 丢弃
- `REJECT-NO-DROP`: 拒绝（不 drop）
